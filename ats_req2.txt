招聘岗位管理系统需求规格说明书

1. 项目背景与目标
某单位人事部计划建设一套招聘岗位管理系统，用于统一管理和发布招聘岗位信息。项目要求通过 AI 智能 编码助手 完成绝大部分代码生成与优化，尽量减少手工编码，重点锻炼需求分析、系统设计及与 AI 协作 的能力。 系在此统基需提础供上完可扩整展的后审批端、R统ES计Tf、ul生A命PI周与期前管端理页等面高，级实功现能招。聘岗位的增删改查，并支持 Excel 批量导入。

2. 用户角色
角色 招聘管理员 审批人（扩展） 普通访客/查看者

职责 创建、编辑、删除、批量导入岗位；查看岗位列表与详情；发起审批。 对处于“待审批”状态的岗位进行审批（通过/驳回）。 可浏览已发布的岗位（无需登录，视扩展需求而定）。

3. 功能需求
3.1 核心功能（必做）
岗位增删改查（CRUD） 通过 Web API 完成对岗位信息的创建、查询、修改和删除。 前端管理页面 PC 端 Web 页面，可调用后端 API 进行岗位的添加、编辑、删除、列表展示和搜索。 Excel 批量上传 前端提供 Excel 文件上传功能，后端解析文件并将岗位数据批量写入数据库。Excel 模板至少包含“岗 位名称”“岗位描述”两列（可扩展）。
3.2 扩展功能（选做，建议至少完成 1-2 项）
岗位审批功能 岗位创建后默认状态为“待审批”，审批人可通过接口审批（通过/驳回），审批通过的岗位方可“发 布”。 岗位统计功能 提供统计接口，如按岗位状态、发布时间等维度统计数量，前端以图表或数字展示。

4. 推荐技术规范

4.1 后端
框架：Spring Boot 接口风格：RESTful Web API 数据格式：统一使用 JSON 数据库：可选用 MySQL 开发方式：必须借助 AI 编码助手完成代码生成和优化

4.2 前端
类型：PC 端 Web 页面 技术选 型：不限，建议使用 Vue 3 + Element Plus，或简单 HTML + CSS + JS（需能调用后端 API） 功能：岗位表单、列表展示、Excel 上传组件、基础交互
